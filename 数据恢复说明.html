📋 Flowy Todo 数据恢复说明

数据存在浏览器本地（localStorage），丢失后可按下面步骤尝试恢复或避免再次丢失。

为什么数据会没？

  • 换了打开方式：用 file:///.../index.html 和用 http://localhost/... 是两套不同的存储，数据不互通。
  • 换了浏览器或配置：不同 Chrome 配置（头像）、无痕模式、其他浏览器，各自一份 localStorage。
  • 清理了网站数据：在浏览器里清理了该站点的“Cookie 和网站数据”，会清空 localStorage。

1️⃣ 如果你有备份 JSON 文件

打开 index.html（用你平时用的同一种方式打开），点击顶栏的 「📤 导入」，选择你的备份 JSON 文件即可恢复。

你当前有一份今天的备份在：~/Downloads/flowy-recovered-2026-02-25.json，里面至少包含项目列表，可以先导入把项目恢复回来。

2️⃣ 从硬盘尝试恢复（任务/便签）

若之前用 Chrome 打开过本应用，数据可能还在 Chrome 的 LevelDB 里。在项目目录运行：

python3 recover_data.py

脚本会扫描所有 Chrome 配置，把找到的 flowy 数据写到桌面 flowy-recovered.json。若生成了该文件且不是空对象，再用上面的「📤 导入」选择该文件恢复。

3️⃣ 以后如何避免再丢？

  • 固定一种打开方式：始终用同一种方式打开（例如始终用 VS Code Live Server 或始终用 file://），不要混用。
  • 定期导出备份：在应用里点「📥 导出」或「下载完整备份」，把 JSON 存到网盘或本机其他目录。
  • 本应用没有服务器，数据只在你这台电脑的浏览器里，备份文件是唯一可恢复来源。

相关文件

  • index.html — 主应用（导入/导出在顶栏）
  • check-data.html — 检查当前浏览器里是否有 flowy 数据
  • extract-data.html — 从当前页面提取并下载备份
  • recover_data.py — 从 Chrome 各配置的 LevelDB 尝试恢复
