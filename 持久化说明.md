# Flowy Todo 本地持久化说明

数据保存在项目目录下的 **`data.json`**，不再依赖浏览器 localStorage，避免因 file:// 协议或清理缓存导致数据丢失。

---

## 一、需要的新建/修改文件

| 文件 | 说明 |
|------|------|
| `server.js` | 已重写为极简版，只读写 `data.json`（已去掉 sql.js） |
| `package.json` | 已去掉 sql.js，仅保留 express、cors |
| `index.html` | 仅数据层调整：从后端加载/保存，`loadProjects`/`saveProjects` 走后端，保存失败有提示 |
| `data.json` | 首次保存时自动生成，无需手建 |

---

## 二、终端命令（首次或重装依赖时执行）

```bash
cd /Users/julianhuang/my-todo
npm install
npm start
```

（请分两行执行，不要在同一行写注释，否则 npm 可能报错。）

- `npm install`：安装 express、cors（若已安装可跳过）
- `npm start`：启动本地服务器，端口 **5999**（避免与 3006 等其它项目冲突）
- 若 5999 也被占用，可设置环境变量换端口：`FLOWY_PORT=3100 npm start`

---

## 三、使用方式

1. 启动服务器后，在浏览器打开：**http://localhost:5999**
2. 不要用 `file:///.../index.html` 打开，否则读写的是浏览器缓存，不会写入 `data.json`。
3. 每次增删改任务/便签/会议/标签/项目后，会自动请求 `POST /api/data/:key`，服务器会覆盖写入 `data.json`。

---

## 四、后端 API（供参考）

- **GET /api/data**  
  返回当前全部数据（如 `flowy-todos`、`flowy-memos` 等），用于页面加载。

- **POST /api/data/:key**  
  请求体：`{ "value": "字符串或 JSON 字符串" }`。  
  服务器读取 `data.json`，更新对应 key，再写回文件。

- **GET /api/health**  
  健康检查，返回 `dataFile` 路径。

---

## 五、异常与提示

- **加载失败**：弹窗提示“加载数据失败”，若当前是 file:// 打开，会建议改用 http://localhost:5999。
- **保存失败**：页面底部约 5 秒红色提示“保存失败，已降级到浏览器缓存”，同时该次修改会写入 localStorage，避免丢数据。

---

## 六、可选：不把数据提交到 Git

若希望 `data.json` 仅保留在本地、不提交到仓库，可在项目根目录创建或编辑 `.gitignore`，加入一行：

```
data.json
```

---

## 七、从旧备份恢复

若你有之前的备份 JSON（如 `flowy-backup-*.json`），可把其中的 `data` 对象按 key（如 `flowy-todos`、`flowy-memos`）复制进 `data.json`，或使用应用内的「📤 导入」选择该备份文件恢复。
