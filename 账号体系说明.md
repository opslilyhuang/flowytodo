# Flowy Todo - 多用户账号体系

## 🎉 新功能：支持多用户注册登录

### ✨ 核心特性

1. **👤 多用户注册/登录** - 每个人独立账号
2. **🔒 数据完全隔离** - 每个用户的数据独立存储
3. **💾 永久保存** - 数据存储在服务器，不会丢失
4. **🔐 安全加密** - 密码使用 SHA-256 加密存储

---

## 🚀 使用方法

### 1. 启动服务器

```bash
cd /Users/julianhuang/my-todo
node server.js
```

服务器启动后会显示：
```
╔═══════════════════════════════════════════╗
║   Flowy Todo 服务器已启动                  ║
║   ✨ 支持多用户账号体系                     ║
║   🌐 打开: http://localhost:5999       ║
╚═══════════════════════════════════════════╝
```

### 2. 注册账号

1. 打开浏览器访问：`http://localhost:5999/login.html`
2. 点击"注册"标签
3. 输入：
   - **用户名**（至少3个字符）
   - **密码**（至少6个字符）
   - **确认密码**
4. 点击"注册"按钮

### 3. 登录

1. 在登录页面输入用户名和密码
2. 点击"登录"
3. 登录成功后自动跳转到主应用

### 4. 开始使用

现在你可以：
- ✅ 添加任务、便签、会议
- ✅ 所有数据自动保存到服务器
- ✅ 数据与其他用户完全隔离

## 📁 文件结构

```
my-todo/
├── server.js              # 后端服务器（带认证）
├── users.json             # 用户账号信息（加密的密码）
├── user_data/             # 用户数据目录
│   ├── 用户A.json         # 用户A的所有数据
│   ├── 用户B.json         # 用户B的所有数据
│   └── ...
├── login.html             # 登录/注册页面
├── index.html             # 主应用（需要登录）
└── package.json           # 依赖配置
```

---

## 🔐 安全特性

- **密码加密**：使用 SHA-256 哈希存储，不保存明文密码
- **Token 认证**：登录后生成 token，24小时自动过期
- **会话管理**：服务器重启后 session 清空，需重新登录
- **数据隔离**：每个用户只能访问自己的数据

## 🆕 账号体系 API

### 注册
```bash
POST /api/auth/register
Content-Type: application/json

{
  "username": "用户名",
  "password": "密码"
}
```

### 登录
```bash
POST /api/auth/login
Content-Type: application/json

{
  "username": "用户名",
  "password": "密码"
}

返回：
{
  "success": true,
  "token": "会话token",
  "username": "用户名"
}
```

### 获取数据（需要 token）
```bash
GET /api/data
Authorization: Bearer <token>

返回：
{
  "success": true,
  "data": {
    "flowy-todos": "...",
    "flowy-memos": "...",
    ...
  }
}
```

## 📊 数据隔离说明

每个用户的数据独立存储在：
```
user_data/<用户名>.json
```

例如：
- 用户 "alice" 的数据在 `user_data/alice.json`
- 用户 "bob" 的数据在 `user_data/bob.json`

**完全隔离，互不影响！**

## 🔄 从旧版本迁移

如果你之前使用过旧版本（localStorage 版本）：

1. **旧数据已无法恢复** - localStorage 已被清空
2. **重新注册账号** - 创建新账号
3. **重新添加数据** - 在新账号下添加你的任务

## ⚙️ 常见问题

### Q: 忘记密码怎么办？
A: 密码使用 SHA-256 加密存储，无法找回。需要重新注册账号或联系管理员。

### Q: 可以修改密码吗？
A: 当前版本不支持修改密码，可以手动编辑 `users.json` 文件（不推荐）。

### Q: 数据会丢失吗？
A: 不会。数据存储在 `user_data/<用户名>.json` 文件中，永久保存。

### Q: 可以导出数据吗？
A: 可以。使用应用内的"导出"功能下载备份文件。

### Q: Session 过期了怎么办？
A: Token 24小时后过期，需要重新登录。
